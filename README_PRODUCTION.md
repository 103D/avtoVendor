# Lassio Web Interface - Production Version

Это чистая пользовательская версия веб-интерфейса для работы с файлами накладных.

## О версии

- **Все отладочные логи удалены** - интерфейс работает чисто без лишней информации
- **Сессии сохраняются** - ваши данные между сеансами
- **Безопасность** - никакой лишней информации не передается

## Структура

```
web_interface_production/
├── app/                    # Backend на Flask
│   ├── __init__.py        # Инициализация приложения
│   ├── routes.py          # API маршруты
│   ├── session_manager.py # Управление сеансами
│   ├── transform_handler.py # Обработка файлов XLSX/DOCX
│   └── __pycache__/       # Кеш Python (создаётся автоматически)
├── static/                # Статические файлы
│   ├── css/              # Стили
│   └── js/               # JavaScript (очищен от логов)
├── templates/            # HTML шаблоны
├── sessions/             # Хранилище сеансов пользователей
├── uploads/              # Загруженные файлы
├── requirements.txt      # Зависимости Python
└── run.py               # Точка запуска приложения
```

## Установка и запуск

### 1. Установить зависимости
```bash
pip install -r requirements.txt
```

### 2. Запустить приложение
```bash
python run.py
```

Приложение будет доступно по адресу: `http://127.0.0.1:5000`

### Для Windows:
```bash
python run.py
# или через start.bat если есть
```

## Функции

### Этап 1 - Загрузка и обработка файла
- Загрузите Excel (.xlsx, .xls, .xlsm) или Word (.docx) файл
- Система автоматически извлечёт SKU, названия товаров и количества
- Отредактируйте количества если нужно
- Сохраните данные и перейдите на Этап 2

### Этап 2 - Синхронизация с API
- Введите учётные данные для API Safia
- Загрузите товары с сервера
- Отредактируйте количества
- Отправьте обновления на сервер

## Поддерживаемые форматы файлов

- **Excel**: .xlsx, .xls, .xlsm
- **Word**: .docx

## Примечания

- Все сеансы сохраняются локально в папке `sessions/`
- Загруженные файлы хранятся в папке `uploads/`
- Сеансы автоматически очищаются после завершения работы
- Данные конфигурации сохраняются в localStorage браузера

## Отличия от development версии

- Все console.log логи удалены из JavaScript
- Все sys.stderr логи удалены из Python
- Удалены файлы резервных копий (.backup)
- Удалена папка .git
- Очищена папка uploads

## Помощь

Если возникают проблемы, проверьте:
1. Установлены ли все зависимости: `pip install -r requirements.txt`
2. Порт 5000 не занят другим приложением
3. Python версия 3.8 или выше
