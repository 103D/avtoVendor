# 🔄 Режим "Экзорд" - Документация

## Что это такое?

**Экзорд** - это специальный режим обработки накладных, который ищет и использует столбец с заголовком **"Отправлено"**.

## Как это работает?

### Обычный режим (по умолчанию):
- Система берёт **первое числовое значение** после названия товара
- Пропускает пустые ячейки и прочерки (-)

### Режим "Экзорд" (при включенном чекбоксе):
1. Система ищет столбец с заголовком **"Отправлено"** (или "otpravleno")
2. Проверяет, есть ли в этом столбце хотя бы одно число  
3. **Если числа найдены** → использует значения из этого столбца
4. **Если столбец пустой или только "-"** → берёт первое число после названия (как в обычном режиме)

## Когда это нужно?

Используйте режим "Экзорд" когда:
- В вашей накладной есть столбец "Отправлено" с количествами
- Этот столбец содержит актуальные данные (не пустой)
- Нужно игнорировать другие числовые столбцы

## Пример

### Структура накладной:
```
| SKU    | Название       | Цена | Склад | Отправлено |
|--------|----------------|------|-------|------------|
| F01234 | Капучино       | 150  | 20    | 5          |
| F05678 | Латте          | 180  | 15    | -          |
```

### Результаты:

**С режимом Экзорд (включен):**
- Капучино: **5** (взято из столбца "Отправлено")
- Латте: **180** (столбец "Отправлено" пустой → взято первое число после названия)

**Без режима Экзорд (выключен):**
- Капучино: **150** (первое число после названия)
- Латте: **180** (первое число после названия)

## 📁 Где хранятся логи изменений?

Все изменения количества товаров записываются в файл:

```
sessions/{ID_сессии}/temp/changes_history.json
```

### Пример реального пути:
```
C:\Users\Safia\Desktop\Diyar\web_interface_production1\sessions\session_1766713809337_hesc03y5c\temp\changes_history.json
```

### Структура JSON файла:
```json
{
  "changes": [
    {
      "дата_время": "25.12.2024 14:35:22",
      "наименование": "Капучино",
      "по_факту": "5",
      "по_накладной": "3",
      "sku": "F01234"
    }
  ]
}
```

### Поля в записи:
- **дата_время** - момент внесения изменения
- **наименование** - название товара
- **по_факту** - количество которое было на самом деле
- **по_накладной** - количество которое было в системе до изменения  
- **sku** - артикул товара

### Как найти файл:
1. Откройте папку `sessions` в корне приложения
2. Найдите папку с нужной сессией (по дате/времени в названии)
3. Откройте внутри папку `temp`
4. Файл называется `changes_history.json`

### Доступ к истории через API:
```javascript
// GET запрос
fetch('/api/get-changes-history?session_id=YOUR_SESSION_ID')
```

### Важно:
- ❌ Файлы накладных **НЕ сохраняются** на сервере (удаляются после обработки)
- ✅ Записываются **только изменения** (где по_факту ≠ по_накладной)
- ✅ История сохраняется для **каждой сессии** отдельно
- 🔒 Данные хранятся локально в папке sessions

## Как использовать на практике?

1. Загрузите накладную как обычно
2. **Поставьте галочку** "🔄 Режим 'Экзорд'" если в накладной есть столбец "Отправлено"
3. Нажмите "⚙️ Обработать"
4. Система автоматически найдёт и использует столбец "Отправлено"

## Технические детали

### Затронутые файлы:
- `templates/stage1.html` - UI с checkbox
- `static/js/stage1.js` - передача флага на сервер
- `app/routes.py` - прием флага и передача в обработчик
- `app/transform_handler.py` - логика поиска столбца "Отправлено"

### Обработка:
- Для **Excel** (.xlsx, .xls, .xlsm): поиск "Отправлено" в заголовках через pandas
- Для **Word** (.docx): поиск "Отправлено" в первых 5 строках таблицы через python-docx
- Оба формата поддерживают режим Экзорд

### Логирование:
При включенном режиме в консоли будет:
```
🔵 Режим Экзорд: ВКЛЮЧЕН
🔄 ЭКЗОРД: Найден столбец "Отправлено" с индексом 4
🔄 ЭКЗОРД: Столбец содержит числа, используем его значения
```

## Важные замечания

⚠️ **Внимание:**
- Режим работает **только при обработке файла**
- Ищется столбец с заголовком содержащим слово "отправлено" (регистр не важен)
- Столбец должен содержать хотя бы одно число (не только "-")

✅ **Совет:**
- Если в накладной есть столбец "Отправлено" - включайте Экзорд
- Если столбца "Отправлено" нет - используйте обычный режим
- Система автоматически вернётся к обычной логике, если столбец "Отправлено" пустой
